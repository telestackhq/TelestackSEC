# Contributing to TelestackSEC

First off, thank you for considering contributing to TelestackSEC! It's people like you that make TelestackSEC such a robust and secure tool.

## Where do I go from here?

If you've noticed a bug or have a question that doesn't belong on the issue tracker, feel free to reach out via email: [hello@telestack.dev](mailto:hello@telestack.dev)

### Found a Bug?
* **Ensure the bug was not already reported** by searching on GitHub under [Issues](https://github.com/telestackhq/TelestackSEC/issues).
* If you're unable to find an open issue addressing the problem, open a new one. Be sure to include a title and clear description, as much relevant information as possible, and a code sample or an executable test case demonstrating the expected behavior that is not occurring.

### Missing a Feature?
* You can request a new feature by submitting an issue to our GitHub Repository. Provide a clear and detailed explanation of the feature you want and why it's important to add.

## Setting Up the Development Environment

1. Fork the repository and clone your fork.
2. Run `npm install` to install all dependencies.
3. Run `npx prisma generate` to generate the Prisma client.
4. Run `npm build` to compile the TypeScript code.

### Running Tests
To run tests, simply run:
```bash
npm test
```

## Pull Request Process

1. Ensure any install or build dependencies are removed before the end of the layer when doing a build.
2. Update the README.md with details of changes to the interface, if applicable.
3. You may merge the Pull Request in once you have the sign-off of other developers, or if you do not have permission to do that, you may request the reviewer to merge it for you.

## Code of Conduct
Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms.
